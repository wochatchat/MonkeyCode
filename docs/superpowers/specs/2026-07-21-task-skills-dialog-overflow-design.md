# 任务技能弹窗横向溢出修复设计

## 背景

任务详情页右上角依次提供“技能、终端、文件、预览、发布”入口。点击“技能”后，`TaskSkillsUpdateDialog` 展示当前任务可选技能。当技能标签数量较多时，弹窗内容会突破白色弹窗右边界，导致描述、标签栏、技能列表和底部按钮延伸到遮罩区域。

## 根因

`DialogContent` 使用 CSS Grid。技能选择区域是其直接 Grid 子项，默认 `min-width: auto` 会使用内部内容的最小宽度参与 Grid 轨道计算。

`TaskSkillPickerBody` 中的 `TabsList` 继承通用样式 `inline-flex w-fit`，标签项使用 `whitespace-nowrap`。大量标签形成较大的最小内容宽度，最终撑宽 Dialog 的 Grid 轨道。弹窗外框仍受 `sm:max-w-xl` 限制，因此 Grid 内容、标题区域和 Footer 在视觉上向右溢出。

## 修复范围

- 在 `TaskSkillsUpdateDialog` 的技能区域直接容器上释放最小内容宽度，并限制内容位于弹窗宽度内。
- 在 `TaskSkillPickerBody` 根节点和标签行上增加可收缩宽度约束。
- 为该选择器中的 `TabsList` 使用父容器可用宽度，保留现有横向滚动行为。
- 保持通用 `Dialog` 和通用 `TabsList` 默认样式不变，控制全站回归范围。
- 保持技能加载、搜索、标签筛选、选择、保存和滚轮行为不变。

## 布局行为

- 弹窗宽度继续使用 `sm:max-w-xl`。
- 标题和描述在弹窗宽度内正常换行。
- 标签保持单行展示，超出可用宽度时在标签栏内部横向滚动。
- 左右标签滚动按钮保持可见且可用。
- 技能列表保持固定高度和纵向滚动。
- Footer 始终位于弹窗右下角。
- 窄屏下弹窗继续使用 `max-w-[calc(100%-2rem)]`。

## 回归测试

- 增加针对技能弹窗宽度约束的源代码回归测试，覆盖直接 Grid 子项、选择器根节点、标签行和标签列表的关键布局类。
- 运行现有任务技能搜索与滚轮测试，确认交互行为保持稳定。
- 运行前端 ESLint。
- 运行 TypeScript 与 Vite online build。

## 独立环境验证

- 从最新 `origin/main` 创建独立 worktree 和修复分支。
- 在独立 worktree 中安装或复用 pnpm 依赖并完成测试与构建。
- 使用独立端口启动 Vite 开发服务器。
- 通过平台预览地址打开任务详情页，点击右上角“技能”。
- 使用包含大量技能标签的数据验证弹窗边界、横向标签滚动、纵向技能滚动和 Footer 位置。
- 分别检查桌面宽屏和窄屏布局。

## 验收标准

- 弹窗所有可见内容均位于白色弹窗边界内。
- 大量标签不会改变弹窗外框宽度。
- 标签横向滚动和左右滚动按钮工作正常。
- 技能列表纵向滚动、搜索、筛选、选择和保存工作正常。
- 桌面与窄屏布局均无横向溢出。
- 定向测试、ESLint 和 online build 全部通过。
